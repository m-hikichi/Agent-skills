# デザイン参照プレイブック

## 第一原則

- スライドを書き始める前に `.slide-work/design-system.yaml` を作る
- タイポグラフィ、余白、色の強調、レイアウト型の使い方を明示的に定義する
- 手を加えていないデフォルトの Marp スタイルを最終成果物として出さない

## まず同梱のスターターを使う

- 既定の組み込みデザイン基盤として `templates/presentation-starter.md` から始める
- `design-system.yaml` を埋める前に、その frontmatter、スタイルブロック、class の語彙、レイアウトの基本要素を読む
- 配色、タイポグラフィ、header/footer の扱い、ページ番号の扱い、コンテンツのリズムを再利用する
- 再利用するのは見た目の方向性であり、プレースホルダーの文面ではない

## 既存の Marp デッキを再利用する

1. 参照デッキの frontmatter を読む
2. theme、スタイル、色、余白、見出しのパターンを抽出する
3. ピクセル単位で複製するのではなく、方向性を再利用する

## PPTX のスタイルを再利用する

1. 色、タイポグラフィ、余白、装飾を観察する
2. Marp で無理なく再現できる部分だけを作り直す
3. 見た目を真似ることより可読性を優先する

## デザイン参照がない場合

- このフォールバックを使うのは、明示的な参照デッキがなく、同梱スターターも依頼に合わない場合だけにする
- 資料タイプと対象読者から始めて、そこから `design-system.yaml` を埋める
- よりステージプレゼン向きの強い印象が必要なら `gaia` を使う
- よりクリーンでエディトリアルな印象が必要なら `uncover` を使う
- 下書きを完成させる前に custom CSS または theme token を追加する

## 色使いの原則

### アクセントカラーの役割分担

同梱スターターでは以下の色が定義されている。各色には意味があり、一貫して使い分ける:

- **accent（シアン）**: 主役の色。タイトル下線、アイコンボックス、主要カードの左ボーダー、リンクに使う。デッキの中で最も多く現れるアクセント色
- **accent-gold（ゴールド）**: 注意喚起・ハイライト。現在フェーズ、推奨アクション、重要な補足に使う。シアンと並置すると視覚的コントラストが生まれる
- **accent-mint（ミント）**: ポジティブな変化。改善、達成、成功、AFTER側の比較に使う
- **accent-warn（レッド系）**: 注意・リスク。課題、リスク、BEFORE側の比較、改善が必要な指標に使う
- **accent-purple（パープル）**: 補助的な区別。第3の分類、中立的な選択肢、技術的な要素に使う

### 色の使用量

- アクセントカラーはスライド面積の **10〜15%** に抑える。多すぎると強調が効かなくなる
- 1スライドで使うアクセントカラーは **最大3色**。4色以上は散漫になる
- 背景は暗色で統一し、テキストは明色にする（ダークテーマの鉄則）
- 同じ意味に同じ色を使い続ける（ポジティブ=mint、ネガティブ=warn を途中で入れ替えない）

## タイポグラフィ階層ガイド

### 3つの視覚的レイヤー

| レイヤー | 用途 | サイズ感 | 色 |
|---------|------|---------|-----|
| タイトル（h2） | 各スライドのtakeaway | 最大・太字 | text-main |
| サブヘッド（h3） | カードやパネルの見出し | 中間・太字 | accent-soft |
| 本文 | 箇条書き、説明文 | 標準 | text-sub |
| 補足 | 出典、注釈、フッター | 小さめ | text-muted |

### タイポグラフィの原則

- タイトルと本文のサイズ差を十分に取る（1.5倍以上）。差が小さいと階層が崩れる
- 太字（`<strong>`）は1スライドに2〜3箇所まで。多用すると何も強調していないのと同じ
- 数字を強調する場合は太字だけでなく、big-stat や metric コンポーネントで視覚的サイズを変える
- 同じスライド内で3種類以上のフォントサイズを使わない（タイトル・本文・補足の3層が上限）

## 余白の設計思想

### なぜ余白が重要か

- 余白は「使い切れなかったスペース」ではなく、**情報を際立たせるためのデザイン要素**
- 余白がない資料は「読む気が起きない壁」に見える
- 十分な余白がある資料は「整理されていて信頼できる」印象を与える

### 余白の目安

- スライド全体の **30〜40%** は余白にする
- タイトルと本文の間: 十分なgap（h2のmargin-bottomで確保済み）
- カード間: 16〜20pxのgap
- スライド端からコンテンツまで: 80px程度のpadding
- 情報ブロック間の区切りが視覚的にわかること

### 「詰め込みたい衝動」への対処

- 1枚に入りきらない → 2枚に分割する。これは妥協ではなく正しい判断
- 文字を小さくして詰め込むのは最悪の選択。可読性が下がり、聞き手が離脱する
- 補足情報は `.quiet-note` や `.micro` で控えめに配置するか、付録に回す

## 視線誘導の原則

### 視線の自然な流れ

- 聞き手の視線は **左上 → 右上 → 左下 → 右下** のZ字型で動く
- 最も重要な情報（タイトル、結論）は左上に配置する
- 補足情報（出典、注釈）は右下に配置する

### 視覚的な「入口」を作る

- 各スライドに **視覚的な入口**（最初に目が行く場所）を1つ作る
- big-stat の大きな数字、カードのアイコン、比較パネルのラベルなどが入口になる
- 入口がないスライド（均一なテキストの壁）は、聞き手がどこから読めばいいかわからない

### 強調のコントラスト

- 強調は **周囲との差** で生まれる。すべてが太字なら何も太字でない
- 1スライドで視覚的に目立つ要素は **最大2つ**（タイトル + 1つのキーポイント）
- big-stat と太字テキストを同時に使うと、視線が分散して強調が効かなくなる

## データ表示のベストプラクティス

### 数字の見せ方

- **1つの数字がメッセージ** → `big-number` archetype で数字を主役にする
- **3〜4つのKPI** → `.metrics-row` で並べ、変化の方向（↑↓）を添える
- **2つの数字の対比** → `two-column-compare` でBefore/Afterとして見せる
- **多数の数字の比較** → テーブルにまとめ、最重要セルだけ太字にする

### テーブルの使い方

- テーブルは **3列以上の構造化された比較** に最適
- 4列×5行を超えるテーブルはスライドに収まらない。分割するか付録に回す
- 最重要セルだけを太字にし、聞き手の注目を誘導する
- テーブルは必ず見出しの直後に配置し、「この表で何を読み取ってほしいか」をタイトルで示す

### グラフが使えない場合の代替

Marp では動的なグラフを埋め込めないため、以下の代替手法を使う:

- 数字 + 方向（↑↓）+ 色で変化を示す（`.metric .change`）
- `two-column-compare` で Before/After を視覚的に対比する
- `process-flow` でデータの変遷を段階的に見せる
- 最も印象的な1つの数字を `big-stat` で大きく見せ、コンテキストをカードで添える
