# Spec Impact Analysis

この skill は「変更前の判定役」である。主な仕事は、変更が `あり` / `なし` / `既存仕様なし` のどれに当たるかを決め、親 skill または利用者が後続の工程を判断できるように判定結果を返すことだ。判定以外の責務はここに持ち込まない。

**判定結果が「既存仕様なし」の場合**: 事実を明示するだけで終わらず、Claude が自動的に `reverse-spec` 相当の最小 bootstrap（requirements.md の最小版 + SPEC-XXX + マトリクス初期行の逆生成）に進む。人間に「仕様を作ってください」と手動作業を要求しない。

## この skill が担当すること

- 変更対象のコード、テスト、仕様、マトリクスを洗い出す
- 仕様影響あり / なし / 既存仕様なし を判定する
- 影響がある場合、どの成果物を更新すべきかを列挙する
- 影響がない場合、その理由を短く残す
- **既存仕様が見つからない場合、reverse-spec 相当の bootstrap を自動実行する**

## この skill が抱え込まないこと

- `requirements.md` や `SPEC-XXX` の本格的な更新（bootstrap 以上の設計作業）
- approved な仕様からのコード実装
- 最終整合性監査の close out
- 判定後に何を進めるかの判断（bootstrap を除く）

## 手順

1. `spec-config.json`、既存の `requirements.md`、`SPEC-XXX`、トレーサビリティマトリクスを確認し、変更対象がどの仕様にぶら下がっているかを探す。
2. [references/change-impact-checklist.md](references/change-impact-checklist.md) を使って、API、UI、入力制約、エラー条件、権限、データ構造、性能保証、rename / move の有無を確認する。
3. 次のどれかで判定する。
   - `あり`: 仕様、要件、テスト、マトリクスのいずれかを更新する必要がある
   - `なし`: 外部挙動やトレーサビリティ契約は変わらない
   - `既存仕様なし`: 対応する仕様が見つからない
4. `あり` の場合は、対象の `FC-` / `NF-`、`SPEC-`、`FR-` / `NFR-`、テストID、マトリクス行と、必要な更新内容を列挙する。
5. `なし` の場合も、「なぜ仕様更新不要か」を 1 文で残す。
6. **`既存仕様なし` の場合**: 以下を Claude が自動で実行する。
   - 変更対象のエントリポイントと触るファイル範囲を確定する
   - 実装コード、既存テスト、README から観測可能な振る舞いを抽出する
   - `requirements.md` の最小版に要件を追加する
   - 最低限の `SPEC-XXX`（契約表・テスト仕様含む）を作成する
   - トレーサビリティマトリクスに初期行を追加する
   - 推測と事実を分け、未確定事項を open questions として残す

## 返答テンプレート

```markdown
## 仕様影響判定

- 判定: あり
- 対象要件: FC-04-04-001, NF-003
- 対象仕様: SPEC-004, FR-002, NFR-003
- 対象テスト: TC-009, TC-010
- 対象マトリクス: `docs/traceability-matrix.md` の FC-04-04-001 / SPEC-004 行
- 理由: 入力バリデーションと API エラーレスポンスが変わり、実装シンボル名も変わるため
- 必要な更新:
  - `requirements.md` の対応表
  - `SPEC-004` の該当節
  - テストケースとトレーサビリティマトリクス
```

```markdown
## 仕様影響判定

- 判定: なし
- 対象要件: FC-04-04-001
- 対象仕様: SPEC-004, FR-002
- 対象テスト: TC-009
- 対象マトリクス: `docs/traceability-matrix.md` の FC-04-04-001 / SPEC-004 行
- 理由: ログ整形と内部関数分割のみで、外部挙動とトレーサビリティ契約は変わらないため
- 必要な更新:
  - なし
```

```markdown
## 仕様影響判定

- 判定: 既存仕様なし
- 理由: 変更対象に対応する `requirements.md`、`SPEC-XXX`、テストID、マトリクス行を特定できなかったため
- 自動 bootstrap を実行:
  - 起こした要件: `docs/requirements.md` に FC-05-01-001 を追加
  - 起こした仕様: `SPEC-015`
  - 起こしたマトリクス行: `FC-05-01-001 -> SPEC-015 -> FR-001`
  - 未確定事項: なし
```
