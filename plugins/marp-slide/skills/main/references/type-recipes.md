# Type-specific storyboard recipes

資料タイプは出発点であり固定テンプレートではない。該当する節だけ使う。

## Executive decision / proposal

結論 → 判断が必要な理由 → 根拠 → 選択肢 → 効果・費用・リスク → 実行条件 → 求める判断。比較軸を揃え、推奨案だけを有利に見せない。

小さなghost deck例：

1. 限定実験なら、上限費用内で主要な不確実性を検証できる
2. 対象業務の大半は3つの反復作業へ集中している
3. 3案を同じ効果・費用・リスク・撤退性で比べる
4. 推奨案は効果だけでなく停止条件を事前に固定できる
5. 本日は予算上限、対象、判定日を決める

## Executive update

状態と例外 → 目標との差 → 変化理由 → リスクと対応 → 必要な判断 → 次のマイルストーン。順調な詳細より、判断を変える例外を優先する。

小さなghost deck例：

1. 全体は計画内だが、移行対象の1領域だけ2週遅れている
2. 遅れは人員不足でなくデータ品質の再作業から生じた
3. 現行案のままでは次のgateで3週間の遅延へ広がる
4. 範囲を分割すれば品質基準を維持して1週間に抑えられる
5. 今週は分割移行の可否だけを判断する

## Analytical read-ahead / report

結論 → 対象と方法 → 主要発見 → 裏付けデータ → 解釈 → 限界 → 次の検証。事実、解釈、提言を視覚的にも分け、無理な承認CTAを付けない。

小さなghost deck例：

1. 継続差は価格帯より初週行動との関連が大きかった
2. 契約月を揃えた対象で90日継続を比較した
3. 差は全cohortで再現したが、因果効果は示さない
4. 選択biasと欠損のため一般化には上限がある
5. 次は無作為化した支援で因果効果を切り分ける

## Technical training

到達目標 → なぜ重要か → mental model → 手順 → worked example → 失敗例 → 演習 → 持ち帰り。説明量ではなく、学習者が次の一手を選べるかで評価する。

小さなghost deck例：

1. 20分後には観測と推測を分けて次の確認を選べる
2. 最初に固定するのは症状の時刻・範囲・期待値である
3. worked exampleでは変更と依存先を同じ時系列で見る
4. いきなり複数設定を変えると仮説を検証できない
5. 演習では片方を否定できる確認を1つ選ぶ
6. 持ち帰る順序は観測→仮説→最小検証である

## Research

問い → 結論と確信度 → 仮説 → データと方法 → 結果 → 解釈 → 限界・代替説明 → 未解決の問い。観測と因果、探索的分析と事前仮説を区別する。

小さなghost deck例：

1. 試行中に集中度は上がったが、出力差の区間はゼロを含む
2. 48人の試行をbaselineと比較し、推定差と区間を示す
3. 結果は方向が揃わず、単一の成功指標へまとめられない
4. 自己選択・季節性・期待効果が代替説明として残る
5. 次の研究課題は対照群を置いた再現性の確認である

例はタイトルの具体度と論理接続を見るためだけに使い、案件固有の根拠、聞き手、終わり方へ書き換える。
